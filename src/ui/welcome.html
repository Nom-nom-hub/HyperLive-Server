üöÄ Quick Start

  1. Open your project folder in VS Code.
  2. Press Ctrl+Shift+P (Cmd+Shift+P on Mac) and type Advanced Live Server: Start Server.
  3. Your default browser opens with live reload enabled.
  4. Edit and save any HTML, CSS, or JS file‚Äîyour browser updates instantly!
You can also use the status bar button or Quick Actions menu for one-click server control.

üîÑ Live Reload

How it works

  • Watches your project files for changes (HTML, CSS, JS, images, etc.).
  • Automatically reloads the browser tab when you save a file.
  • Supports cache-busting, SPA fallback, and custom reload scripts.

Advanced Usage

  • Configure file types to watch in Settings → Extensions → Advanced Live Server.
  • Enable/disable SPA mode for single-page apps.
  • Use the Restart Server command to reload config without restarting VS Code.
// Example: Custom reload script window.addEventListener('liveServerReload', () => { // Custom logic here });
If live reload isn‚Äôt working, check the Troubleshooting section below.

üì∏ Screenshots

Capture Options

  • Full page, viewport, or specific element (by CSS selector).
  • Choose format: PNG, JPEG, WebP.
  • Set quality, add custom CSS, hide elements, and more.

How to use

  1. Open the page you want to capture in the browser.
  2. Use the Quick Actions menu or run Take Screenshot command.
  3. Choose your options and save the image.
// Example: Capture element { selector: '.header', format: 'png', quality: 90 }
Screenshots are saved to your project folder by default.

üë• Team Collaboration

How it works

  • Start a session and share the session ID with your team.
  • Real-time sync of file changes, server state, and browser preview.
  • See who‚Äôs connected and manage sessions from the status bar or Quick Actions.

How to use

  1. Start a session: Start Collaboration from Quick Actions or Command Palette.
  2. Share the session ID (copy from status bar or session info).
  3. Teammates join using Join Collaboration and the session ID.
  4. Stop the session anytime from the status bar or Quick Actions.
Collaboration is local-first and privacy-friendly. No data leaves your machine unless you use Cloud Preview.

üìä Analytics

What you get

  • Performance stats: uptime, load time, file changes, and more.
  • Usage analytics for your project (local only, unless cloud enabled).
  • Export reports in JSON, CSV, or HTML.

How to use

  1. Open the Analytics Dashboard from Quick Actions or Command Palette.
  2. View stats, trends, and export options.
Analytics are private by default. Enable Cloud Preview to share stats with your team.

Live reload isn‚Äôt working?
Make sure your file is saved, and check that the server is running. Try restarting the server or VS Code. See logs in the Output panel (Advanced Live Server channel).
AI features not working?
Check your AI provider settings and API key. Some providers require a valid key and internet connection.
Collaboration issues?
Ensure all users are on the same network or using Cloud Preview. Share the correct session ID.
